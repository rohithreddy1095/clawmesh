ClawMesh

distributed AI mesh — independent claws that join into one intelligent network

Each device is a claw — an independent mesh node with its own identity, capabilities, and intelligence. Claws discover each other via mDNS, establish Ed25519 trust, and form a mesh where context flows organically and execution routes to whichever node has the right capability.

Mesh Topology
Core Principle
No central coordinator. Each node discovers peers via _clawmesh._tcp mDNS, verifies identity with Ed25519 signed handshakes, and routes messages to whichever peer advertises the needed capability. Local-first — always prefer local capabilities over mesh peers.

Context Engine
Propagation
Gossip Protocol
Sensor observations become ContextFrames — typed data packets that broadcast across the mesh via WebSocket gossip. No polling, no file sync. Nodes build awareness organically.
observation event human_input inference agent_response capability_update
State
World Model
Each node maintains a WorldModel — a live picture of mesh-wide state ingested from context frames. The planner LLM reasons over this emergent knowledge to propose actions.
  • world-model.ts ingest + track state
  • context-propagator.ts broadcast to peers
  • context-types.ts frame type definitions

Intelligence Layer
Planner
Pi-Powered LLM Session
Multi-provider conversational planner using AgentSession from pi-agent-core. Reasons over mesh-wide context, proposes TaskProposals with approval levels, and forwards execution commands to capable peers.
  • pi-session.ts conversational agent loop
  • clawmesh-mesh-extension.ts mesh tools
  • farm-context-loader.ts domain knowledge
Learning
Pattern Memory
Operator approve/reject decisions are captured, correlated to trigger conditions, and stored. After repeated consistent decisions, patterns become mature and gossip across the mesh — collective intelligence.
  • pattern-memory.ts learn + persist
  • Persisted to ~/.clawmesh/mesh/patterns.json
  • Mature patterns broadcast to peers
Trust & Safety
Critical safety rule: LLM-only evidence (evidence_sources: ["llm"]) is hard-blocked from triggering any physical actuation. The trust policy requires sensor or human evidence for L2/L3 operations.
